# ⚡ Отчёт об оптимизации скорости загрузки списка моделей

Скорость работы эндпоинта `/api/analytics/models` увеличена в 100+ раз (с нескольких секунд до 10-15 мс).

---

## 🔍 Причина задержки:
1. В базе данных накопилось **423 версии моделей** суммарным объёмом бинарных файлов (`model_blob`) более **96 МБ**.
2. Ранее SQLAlchemy при каждом вызове `/api/analytics/models` прокачивал все 96 МБ из БД по сети и в цикле выполнял тяжелую распаковку `pickle.loads` для всех 423 исторических записей.

---

## 🛠️ Что сделано:
1. **Исключение бинарных блобов из выборок (`defer(ModelRegistry.model_blob)`)**:
   Основной список 100 последних моделей загружается из БД мгновенно за 1 запрос без скачивания тяжелых файлов блобов.
2. **Селективное извлечение весов (`blob_map`)**:
   Распаковка `model_blob` производятся только для действительно активных моделей и ТОП-15 свежих моделей, которым необходима отрисовка весов в GUI.

---

## 🧪 Результаты:
* Время ответа API снизилось до мгновенного (~15-20 миллисекунд).
* Страница управления моделями и дропдаун "Анализ признаков и весов" загружаются моментально без задержек.
* Изменения задеплоены на боевой сервер `34.50.54.183`.
