# PolyFlip — Архитектура

---

## 1. Обзор

PolyFlip — самодостаточный аналитический микросервис и торговый бот для торговли на Polymarket. Бот самостоятельно собирает данные по 15-минутным рынкам, обучает прогностические модели, анализирует текущие цены и совершает сделки через API Polymarket CLOB.

```
┌─────────────────────────────────────────────────────┐
│                  Дашборд (HTML)                      │
│        Chart.js + Сводка + Журнал торговли          │
├─────────────────────────────────────────────────────┤
│               REST API (FastAPI)                     │
│    /dashboard │ /api/settings │ /api/trading/stats  │
├─────────────────────────────────────────────────────┤
│            Торговый движок (trading/)               │
│  trade_worker_cycle │ расчет PnL │ выполнение сделок│
├─────────────────────────────────────────────────────┤
│             ML + Обучение (models/)                 │
│    LogisticRegression │ ModelTrainer (5-fold CV)    │
├─────────────────────────────────────────────────────┤
│            Слой данных (db/ + PostgreSQL)           │
│  snapshots │ live_markets │ trades │ model_registry │
├─────────────────────────────────────────────────────┤
│               Планировщик (scheduler/)               │
│ collector │ resolver │ retrain │ trade_worker │ backup│
└─────────────────────────────────────────────────────┘
```

## 4. ML-слой (Прогнозирование разворота)

Модель решает задачу бинарной классификации: предсказать вероятность разворота рынка (`flip_vs_final`) к моменту его закрытия на основе текущего состояния.

### 4.1 Признаки (Features)
При сборе данных и инференсе формируется вектор из 6 признаков:
1. `time_left_min` (float) — количество минут до закрытия рынка (0.0 – 15.0).
2. `mid_price` (float) — средняя цена токена YES на Polymarket (обычно от 0.05 до 0.95).
3. `spread` (float) — разница между лучшим ask и bid ценами в стакане.
4. `volume_5min` (float) — объем торгов по данному рынку за последние 5 минут.
5. `price_velocity` (float) — скорость изменения цены за последние 5 минут.
6. `hour_of_day` (int) — час UTC времени создания/наблюдения (0 – 23).

### 4.2 Обучение
Обучение выполняется в [trainer.py](file:///C:/Users/orlov/.gemini/antigravity/scratch/polyflip/polyflip/models/trainer.py):
* Данные загружаются из `market_snapshots` (исключая рынки в статусе `PENDING`).
* Применяется кросс-валидация `StratifiedKFold` (5 фолдов).
* Финальная модель `LogisticRegression` обучается на всей доступной выборке актива и сохраняется в таблицу `model_registry` в бинарном виде (`pickle`).
* Внедрен мониторинг качества: модель активируется только в случае, если ее точность на валидации выше, чем у baseline (простого большинства).

## 5. Торговый движок (Trading Engine)

Вся торговая логика работает локально в фоновом режиме внутри контейнера `scheduler`.

### 5.1 Торговый цикл (`trade_worker_cycle`)
Каждые 60 секунд (настраивается через `LIVE_POLL_INTERVAL_SECONDS`) запускается процесс:
1. **Проверка лимита дневных потерь**: Если суммарный PnL за текущие календарные сутки в UTC достиг лимита (например, -$100.0), торговля блокируется до конца дня.
2. **Получение данных**: Запрашиваются активные рынки из `live_markets` и загружаются активные версии моделей из `model_registry`.
3. **Генерация предсказания**: Для каждого рынка рассчитывается вероятность флипа $P(flip)$ на основе текущих признаков.
4. **Принятие решения**:
   * Если $P(flip) > threshold\_flip$ (модель ожидает разворот) — бот покупает недооцененный токен (NO, если текущая цена YES > 0.5; YES в противном случае).
   * Если $P(flip) < threshold\_no\_flip$ (модель считает, что рынок прав) — бот покупает текущего фаворита.
   * В ином случае сделка пропускается со статусом `SKIPPED` и записью подробной причины в историю.
5. **Исполнение ордера**: Через [trader.py](file:///C:/Users/orlov/.gemini/antigravity/scratch/polyflip/polyflip/trading/trader.py) бот отправляет ордер в Polymarket CLOB. Размер сделки вычисляется динамически (линейное масштабирование от `TRADE_BET_SIZE_USDC` до `MAX_BET_SIZE_USDC` в зависимости от размера Expected ROI (`edge`)).

## 6. Схема базы данных

### 6.1 Таблица `market_snapshots`
Исторические срезы состояний рынков для обучения ML-моделей.
* `id` (BIGINT, PK) — инкрементальный идентификатор.
* `market_id` (VARCHAR) — уникальный ID рынка на Polymarket.
* `asset` (VARCHAR) — криптовалютный тикер базового актива (BTC, ETH и др.).
* `recorded_at` (TIMESTAMPTZ) — время фиксации снапшота.
* `time_to_expiry_min` (FLOAT) — сколько минут оставалось до закрытия.
* `mid_price` (FLOAT) — средняя цена токена YES.
* `spread` (FLOAT) — размер спреда bid/ask.
* `volume_last_5min` (FLOAT) — объем торгов за последние 5 минут.
* `price_velocity` (FLOAT) — скорость движения цены.
* `hour_of_day` (INTEGER) — час UTC.
* `final_outcome` (VARCHAR) — исход резолва рынка (`YES`, `NO`, `INVALID`, `PENDING`).
* `flip_vs_final` (BOOLEAN) — признак совершения разворота относительно цены на момент снапшота.

### 6.2 Таблица `live_markets`
Текущее состояние активных рынков (обновляется раз в минуту парсером).
* `market_id` (VARCHAR, PK) — ID рынка.
* `condition_id` (VARCHAR) — служебный ID для CLOB API.
* `asset` (VARCHAR) — базовый актив.
* `question` (TEXT) — формулировка вопроса.
* `end_time_est` (TIMESTAMPTZ) — дата и время закрытия рынка.
* `yes_token_id` (TEXT) — ID токена YES для торговли.
* `no_token_id` (TEXT) — ID токена NO для торговли.
* `current_yes_price` (FLOAT) — текущая цена YES.
* `current_spread` (FLOAT) — текущий спред.
* `volume_5min` (FLOAT) — объем за 5 минут.
* `price_velocity` (FLOAT) — скорость движения цены.
* `updated_at` (TIMESTAMPTZ) — время последнего обновления.

### 6.3 Таблица `trade_history`
Журнал всех совершенных ботом сделок и пропущенных рынков.
* `id` (SERIAL, PK) — ID сделки.
* `market_id` (VARCHAR) — ID рынка.
* `asset` (VARCHAR) — базовый актив.
* `outcome_bought` (VARCHAR) — что куплено (`YES`, `NO` или `NONE` при пропуске).
* `amount_usdc` (FLOAT) — размер ставки в USDC.
* `executed_price` (FLOAT) — цена исполнения сделки.
* `predicted_flip_prob` (FLOAT) — предсказанная моделью вероятность разворота.
* `active_features` (TEXT) — вектор признаков, при котором сработало решение.
* `model_version` (INTEGER) — версия модели.
* `status` (VARCHAR) — статус (`SUCCESS`, `FAILED`, `SKIPPED`).
* `error_msg` (TEXT) — описание ошибки или причины пропуска.
* `pnl` (FLOAT) — финансовый результат сделки (рассчитывается после резолва).
* `mode` (VARCHAR) — режим работы (`LIVE` или `PAPER`).
* `created_at` (TIMESTAMPTZ) — дата создания записи.

### 6.4 Таблица `model_registry`
Реестр сериализованных ML-моделей.
* `id` (SERIAL, PK).
* `asset` (VARCHAR) — базовый актив модели.
* `version` (INTEGER) — порядковый номер версии.
* `trained_at` (TIMESTAMPTZ) — дата обучения.
* `data_points` (INTEGER) — размер выборки при обучении.
* `accuracy` (FLOAT) — точность на валидации.
* `baseline` (FLOAT) — точность baseline-модели.
* `model_blob` (BYTEA) — сериализованная модель (`pickle`).
* `is_active` (BOOLEAN) — флаг активной модели в проде.

### 6.5 Таблица `runtime_settings`
Динамические настройки системы, редактируемые в дашборде без перезапуска контейнеров.
* `key` (VARCHAR, PK) — имя параметра (например, `TRADING_ENABLED`, `MIN_EDGE`).
* `value` (TEXT) — строковое представление значения.
* `updated_at` (TIMESTAMPTZ) — время обновления.

### 6.6 Таблица `slippage_log`
Журнал проскальзываний, фиксирующий разницу между ожидаемой ценой сделки и фактической ценой исполнения.
* `id` (SERIAL, PK)
* `trade_id` (INTEGER, FK) — ссылка на `trade_history`.
* `market_id` (VARCHAR) — ID рынка.
* `asset` (VARCHAR) — базовый актив.
* `outcome_bought` (VARCHAR) — `YES` или `NO`.
* `expected_price` (FLOAT) — цена, по которой бот собирался купить.
* `executed_price` (FLOAT) — реальная средняя цена исполнения ордера.
* `slippage` (FLOAT) — разница цен.
* `slippage_pct` (FLOAT) — проскальзывание в процентах.
* `bet_size_usdc` (FLOAT) — размер ставки.
* `slippage_cost_usdc` (FLOAT) — финансовая стоимость проскальзывания.
* `mode` (VARCHAR) — режим (`LIVE` или `PAPER`).
* `created_at` (TIMESTAMPTZ) — дата записи.

## 7. Безопасность и надежность

* **Аутентификация API**: Запросы к защищенным эндпоинтам требуют передачи `X-API-Key`. Сравнение выполняется за константное время (`secrets.compare_digest`), исключая timing-атаки.
* **Хранение ключей**: Приватный ключ кошелька в [trader.py](file:///C:/Users/orlov/.gemini/antigravity/scratch/polyflip/polyflip/trading/trader.py) не хранится в памяти в открытом виде — сохраняется только его SHA-256 хеш для обнаружения изменений параметров.
* **Rate Limiting**: Custom Middleware блокирует подозрительную активность, ограничивая частоту запросов с одного IP (60 запросов/мин) и автоматически очищая память от неактивных записей.
* **Сетевая изоляция**: База данных PostgreSQL привязана исключительно к `127.0.0.1:5432`, что полностью закрывает ее от внешней сети. Доступ к БД с хоста осуществляется только через SSH-туннели.
* **Резервное копирование**: По расписанию в `scheduler` запускается `backup_job`, создающий дампы СУБД с ротацией (хранятся последние 7 копий).
