# Инструкция по деплою (Google Cloud e2-small)

Так как код хранится на GitHub, деплой на боевой сервер сводится к скачиванию репозитория и запуску Docker-контейнеров.

## Шаг 1: Подключение к серверу
Подключитесь к вашему Google Cloud серверу по SSH.

## Шаг 2: Клонирование репозитория
Если репозиторий клонируется впервые:
```bash
git clone https://github.com/orlovRoman/flipoly.git
cd flipoly
```

Если репозиторий уже скачан, просто обновите код:
```bash
cd flipoly
git pull origin main
```

## Шаг 3: Настройка окружения
В папке проекта находится файл `.env.example`. Скопируйте его в `.env`:
```bash
cp .env.example .env
```
(При необходимости вы можете отредактировать `.env` через `nano .env`, чтобы изменить пароли).

## Шаг 4: Запуск проекта
Сборка и запуск контейнеров в фоновом режиме:
```bash
docker-compose build
docker-compose up -d
```

## Шаг 5: Проверка
1. Проверьте, что все 3 контейнера (db, api, scheduler) работают:
```bash
docker-compose ps
```
2. Откройте в браузере дашборд: `http://<IP-АДРЕС-ВАШЕГО-СЕРВЕРА>:8001/dashboard`
(Убедитесь, что в правилах Firewall Google Cloud открыт порт `8001` для внешних подключений, если хотите заходить не с локального адреса).

## Перезапуск при обновлениях
Когда мы добавим новые функции (Фаза 5), для обновления вам нужно будет просто выполнить:
```bash
git pull origin main
docker-compose up -d --build
```
